# 电子出版达标评估与改进方案

> 评估日期：2026-06-18
> 评估范围：81 个文件，26,070 行，18 章 + 9 个附录
> 目标标准：专业电子出版（ePub/PDF/HTML 三格式 + 纸质印刷就绪）

## 一、诊断摘要

通过 15 项自动化诊断扫描，发现当前文档工程在以下维度存在差距：

| 维度 | 当前状态 | 达标率 |
|------|----------|--------|
| **元数据完整性** | 1/81 文件含 YAML Frontmatter | **1.2%** |
| **排版一致性** | 行尾空格 95 处，中英文粘连 28+ 处/章 | **~60%** |
| **链接有效性** | 58 个外部 URL 未验证，内部交叉引用存在断链风险 | **~70%** |
| **格式规范性** | 无自动化 Lint/检查，无统一 Style Guide | **0%** |
| **多格式输出** | ePub/HTML 已生成但缺少样式模板 | **~50%** |
| **校稿流程** | 无正式校对记录，无术语一致性检查 | **0%** |
| **出版就绪** | 缺版权页/ISBN/CIP/Colophon | **0%** |

| # | 问题 | 严重度 | 影响范围 | 数据 |
|---|------|--------|----------|------|
| A1 | **YAML 元数据缺失** | 🔴 P0 | 80/81 个文件 | 仅主报告有 `title/date/lang/toc` frontmatter |
| A2 | **术语不一致** | 🟡 P1 | 全报告 | 无术语一致性检查工具 |

## 五、工具清单

| 工具 | 用途 | 安装方式 |
|------|------|----------|
| **markdownlint-cli** | Markdown 格式检查（行尾空格、标题层级、列表风格） | `npm install -g markdownlint-cli` |
| **lychee** | 链接有效性检查（内部交叉引用 + 外部 URL） | `cargo install lychee` 或 `brew install lychee` |
| **vale** | 自然语言风格检查（术语一致性、拼写、可读性） | `brew install vale` 或 `choco install vale` |
| **pangu.py** | 中英文自动空格 | `pip install pangu` |
| **pandoc** | 多格式转换（已安装 v3.10 ✅） | — |
| **mdBook** | 文档站生成（需安装） | `cargo install mdbook` |
| **WeasyPrint** | PDF 排版（CSS 分页媒体） | `pip install weasyprint` |
| **EPUB-Checker** | ePub 验证（W3C 标准） | `npm install -g epubcheck` |
