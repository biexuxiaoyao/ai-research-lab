# 🧭 Agent 决策指南

## 下探决策规则

当主 Agent 审视这份知识地图时，按以下优先级决定下一步：

1. **高价值 + 浅覆盖**：标记为 🔄 [L2] 的节点，总纲已有基础但缺乏深入分析 → **优先下探到 [L3]**
2. **空白区域扫描**：未被标注的潜在研究方向 → **广度扩展优先**（先识别、标记，再决定是否下探）
3. **依赖解锁**：某些 ⬜ 节点的研究依赖前序节点完成 → **等待依赖满足**
4. **时效性触发**：外部事件（新产品发布、重要论文、安全事故）可能提升某节点的优先级 → **动态调整**

## 下探深度判断

- **[L3] → [L4]**：该方向已有 3 层分析，继续下探需要**实证数据、一手访谈、或新的分析框架**。仅当出现这些新信息时才继续
- **[L2] → [L3]**：该方向已有初步覆盖，需要**系统性深挖**（新的搜索、对比分析、案例收集）
- **[L1] → [L2]**：该方向仅有概念定义，需要**首次研究投入**

## 广度扩展判断

- 每隔 1-2 个月扫描一次"未被标注的研究机会"
- 每次深度研究的结论中检查"是否存在新的交叉连接"
- 关注相邻领域（MLOps、LLMOps、AI Safety、Human-Computer Interaction）的溢出效应
