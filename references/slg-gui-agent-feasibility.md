# 探索：SLG GUI Agent 的可行性与技术路线

> 时间：2026-06-13
> 主题：从 UI-TARS-2 出发，探讨训练一个完整 SLG Agent 的个人可行性

---

## 一、UI-TARS-2 分析

### 1.1 是什么

字节跳动 Seed 团队于 2025 年 9 月发布的原生多模态 GUI 智能体。一个统一模型驱动，可自主操作**电脑、手机、浏览器**，甚至玩 15 款 HTML5 小游戏。

### 1.2 关键性能

| 基准测试 | 得分 | 对比 |
|---------|------|------|
| Online-Mind2Web | 88.2 | 优于 Claude/OpenAI |
| OSWorld | 47.5 | 优于 Claude/OpenAI |
| AndroidWorld | 73.3 | 优于 Claude/OpenAI |
| 15 款游戏套件 | 59.8（人类 60% 水平） | 是 OpenAI CUA 的 **2.4x**，Claude Computer Use 的 **2.8x** |
| SWE-Bench Verified | 68.7 | 可执行软件工程修复 |

### 1.3 技术架构

| 维度 | 详情 |
|------|------|
| 基座模型 | Seed-Thinking-1.6，230B 总参（23B 激活 + 532M 视觉编码器），MoE |
| 核心训练范式 | 多轮 RL：CT（持续预训练）→ SFT（监督微调）→ RL（强化学习）循环 |
| 训练环境 | 统一沙盒平台，支持百万级交互 |
| 奖励设计 | 可验证任务用确定性验证器/LLM-as-Judge；不可验证任务用自训练 ORM（F1=83.8） |
| 数据飞轮 | 高质量轨迹 → SFT 数据集，低质量轨迹 → CT 数据集，零浪费 |

### 1.4 人工标注介入

**需要，但方式和规模与传统不同。**

| 阶段 | 标注方式 | 特点 |
|------|---------|------|
| CT 冷启动 | "就地标注"：标注者在自己电脑上操作，有声思维口述，ASR 转录 + LLM 润色 | 低打扰，新手+专家互补 |
| SFT | "交互式标注"：模型在真实环境运行，提出候选动作，标注者接受或修正 | 严格 on-policy，比离线标注效率高 |
| RL | 不需人工标注 | 环境反馈 + ORM 自动奖励 |

## 二、策略游戏的难度层次（Agent 视角）

### 2.1 从棋类到 SLG：本质差异

| 维度 | 象棋/围棋 | 无尽的拉格朗日（SLG） |
|------|-----------|---------------------|
| 信息 | 完全信息 | 不完美信息（战争迷雾） |
| 博弈方 | 2 方零和 | N 方非零和（联盟/背叛） |
| 状态空间 | 离散、有限 | 连续+离散混合，动态变化 |
| 操作界面 | 单一棋盘 | 多层嵌套 UI：基地→舰队→星图→联盟→商店… |
| 时间尺度 | 回合制，秒级 | 实时制，操作链跨度小时到月 |
| 奖励密度 | 密集（每步可评估） | 极度稀疏（赛季末才知排名） |
| UI 深度 | ~3-5 步 | 生产一艘船并采矿 ≈ 15+ 步操作 |

### 2.2 AlphaGo 成功条件的不可复制性

AlphaGo 成功的三个支柱在 SLG 完全不成立：
1. **完全信息** → SLG 有不完美信息
2. **2 方零和博弈** → SLG 是 N 方非零和
3. **自我对弈可行**（3000 万局） → SLG 环境不完全可控，其他玩家不可模拟

## 三、无尽的拉格朗日游戏分析

### 3.1 基本信息

- 网易自研，2021 年 8 月上线，太空科幻 SLG
- 平台：Windows / Android / iOS
- 2026 年 1 月国内更名为《星际猎人》
- 核心：赛季制（星系协议），每赛季基建清零但保留蓝图

### 3.2 核心系统复杂度

| 系统 | 复杂度 |
|------|--------|
| 计划圈 | 8 种指令类型，策略值 150 上限，实时恢复 |
| 基地 | 1-9 级，30+ 可交互建筑入口，三大功能区 |
| 舰船 | 6 个等级 × 数十种型号 × 子型号 × 蓝图强化（技术值加点） |
| 资源 | 3 种基础资源 + 3 种货币 + 策略值/动员令双行动资源 |
| 联盟 | 3 层组织（家族/公会/同盟） + 联合体 + 大规模集结作战 |
| 赛季 | 数月周期，基地清零但保留蓝图/技术值 |

### 3.3 SLG Agent 的核心难点（以本作为例）

1. **奖励稀疏**：赛季末才知排名，信用分配时间窗口长达数月
2. **UI 超复杂**：比 OSWorld 任何应用都复杂 5-10 倍
3. **多时间尺度并行**：秒级战斗微操 ↔ 分钟级采集调度 ↔ 天级建筑规划 ↔ 月级战略
4. **社会博弈**：联盟外交、信息欺诈、囚徒困境 — 需要 Theory of Mind

## 四、个人训练 SLG Agent 的可行性

### 4.1 直接结论：不可行

**完整 SLG Agent** 目前对个人不成立，原因不是"算力不够"，而是：

| 关卡 | 瓶颈 | 个人可解？ |
|------|------|:--:|
| 训练环境 | 真实游戏无 API/无模拟器/有反作弊 | ❌ |
| 奖励设计 | 月级信号 → 分钟级子奖励，每层都是独立研究 | ❌ |
| RL 训练算力 | 7B+ 全参数 RL 需 4-8×A100，云租 ￥5-20 万 | ⚠️ 勉强 |
| 基座模型预训练 | 需要 VLM + 理解复杂的 UI 元素关系 | ⚠️ 可用开源 |
| 多智能体博弈 | 博弈论/multi-agent RL 的前沿难题 | ❌ |

### 4.2 个人训练成本参考（通用 RL Agent，非 SLG 专用）

| 方案 | 硬件 | 租用成本 | 周期 |
|------|------|---------|------|
| MiniMind-3（64M 参数）全程 RL | 单卡 RTX 3090 | **≈￥3** | 2.3h |
| Qwen2.5-VL-7B + LoRA SFT | 1×A100 | ￥500-2000 | 几十小时 |
| Qwen2.5-VL-7B + 全参 RL | 4-8×A100 | ￥5 万-20 万 | 几周 |
| 72B + 全参 RL | 16-32×A100 | ￥30 万-100 万+ | 数周-数月 |

> 关键教训：RL 首次成功率约 20-30%，通常需 3-5 次迭代才能出可用的结果。

### 4.3 个人可做的分层路线

| Tier | 目标 | 预算 | 周期 | 产出价值 |
|------|------|------|------|---------|
| **T1** | VLM 辅助脚本：截图→分析→建议（不操作 UI） | ￥100-2000 | 几天-几周 | 可用工具，学到 prompt engineering |
| **T2** | 简化沙箱 + 单任务 Agent（如"开局到 4 本"） | ￥2-5 万 | 2-3 月 | 论文/博客，概念验证 |
| **T3** | 完整 SLG Agent 原型（需自建环境+层次化 RL） | ￥50 万+ | 6-12 月 | 博士课题/创业级别 |

## 六、结论摘要

1. **UI-TARS-2** 代表了 GUI Agent 从"工具链拼接"走向"统一原生模型"的范式转变，核心驱动力是多轮 RL + 数据飞轮
2. **人工标注仍然需要**，但集中在冷启动和 SFT 阶段做 on-policy 交互式标注，RL 阶段无需人工
3. **象棋/围棋与 SLG 的差异不是复杂度量变，而是问题范式的质变**（完全信息 vs 不完美信息，2 方 vs N 方，回合 vs 实时，密集奖励 vs 稀疏奖励）
4. **个人训练完整 SLG Agent 当前不可行**，瓶颈不在算力而在环境、奖励、博弈论
5. **可行的个人路线**：用现有 VLM 做 SLG 决策辅助工具（T1），这是成本最低且能实际产出的方向
